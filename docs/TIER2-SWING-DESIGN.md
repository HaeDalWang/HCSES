# Tier 2: Short-Swing Alert System — 설계 문서

> "5~15일 내 치고 빠질 수 있는 기술적 반등 구간을 포착한다"

---

## 1. 철학 및 원칙

### 1.1 Tier 1과의 관계

| | Tier 1 (기존 HCSES) | Tier 2 (Swing) |
|--|--|--|
| **질문** | "이 종목을 장기로 들어가도 되는가?" | "여기서 단기 반등이 나올 확률이 높은가?" |
| **근거** | 밸류에이션 바닥 + 모멘텀 전환 | 기술적 과매도 + 반등 시그널 |
| **확신도** | 극도로 높음 (연 2~5회) | 중간 (월 3~10회) |
| **보유 기간** | 60일 타임컷 | 5~15일 타임컷 |
| **손실 한도** | ATR × 2.0 | ATR × 1.5 (짧게 끊음) |
| **목표 수익** | PBR 중앙값 복귀 (+50~100%) | 기술적 저항선 도달 (+5~15%) |

### 1.2 Tier 2 핵심 원칙

1. **"반등의 초기를 잡는다"** — 바닥을 잡는 게 아니라, 바닥을 확인한 직후의 첫 반등을 잡음
2. **"복합 확인"** — 단일 지표로 절대 트리거하지 않음. 최소 2개 이상의 독립적 시그널 동시 충족
3. **"짧게 끊는다"** — 목표가 도달 또는 타임컷, 둘 중 먼저 도달하면 종료
4. **"시장 환경 필터"** — 전체 시장이 폭락 중이면 개별 종목 반등도 믿지 않음
5. **"기존 Tier 1과 중복 불가"** — Tier 1이 이미 발동한 종목은 Tier 2 알림 제외

### 1.3 이 시스템이 잡으려는 구간 (예시)

```
가격
 │
 │    ╭──────╮              ← MA20 (중기 이평)
 │   ╱        ╲
 │  ╱          ╲
 │ ╱            ╲
 │╱              ╲          ← 하락 구간
 │                ╲
 │                 ╲        ← RSI 35 이하 진입
 │                  ╰─╮
 │                    │     ← 볼린저 하단 터치 (지지 확인)
 │                    ╰─┐
 │                      ↑   ← [Tier 2 진입점] 반등 시작 + 거래량 증가
 │                      │
 │                   5~15일 보유
 │                      │
 │                      ↑   ← [목표가] MA20 부근 도달 또는 +10%
 ╰──────────────────────────────── 시간
```

## 3. 스코어링 체계

### 3.1 점수 구조

```
Tier 2 Score = 기술적반등(40) + 가격지지(30) + 거래량확인(30)
Alert Threshold = 70점 이상
```

### 3.2 조건 1: 기술적 반등 초기 (40점)

**충족 조건 (둘 중 하나):**

#### 조건 A: RSI 과매도 탈출
```
RSI_prev ≤ 35 AND RSI_curr > 35
```

- Tier 1의 `RSI_prev ≤ 30 AND RSI_curr > 35`보다 완화 (35 진입 기준)
- RSI 35는 "약한 과매도"로, 단기 반등이 시작되는 빈도가 높은 구간

#### 조건 B: RSI 바닥 반등 (연속 상승)
```
RSI_curr < 40
AND RSI_curr > RSI_prev
AND RSI_prev > RSI_prev_prev (2일 연속 RSI 상승)
```

- RSI가 아직 40 이하이지만 방향이 꺾인 상태
- 절대값이 아닌 "방향 전환"을 포착

**보수 원칙:**
- RSI가 None이면 0점
- RSI 데이터가 2일 미만이면 0점

### 3.3 조건 2: 가격 지지 확인 (30점)

**충족 조건 (둘 중 하나):**

#### 조건 A: 볼린저 하단 반등
```
Low_prev ≤ BB_lower (전일 저가가 볼린저 하단 터치 또는 이탈)
AND Close_curr > BB_lower (당일 종가가 볼린저 하단 위로 복귀)
```

- 볼린저 하단 터치 후 반등 = "과매도에서 평균회귀 시작" 시그널
- 기술적 분석에서 가장 검증된 평균회귀 패턴 중 하나

#### 조건 B: 단기 이평 회복
```
Close_prev < MA5 (전일 단기 이평 아래)
AND Close_curr > MA5 (당일 단기 이평 돌파)
AND Close_curr < MA20 (아직 중기 이평 아래 = 추가 상승 여력)
```

- 단기 이평 회복 = "초단기 추세 전환"
- MA20 아래에 있으므로 목표가(MA20)까지 상승 여력 존재

**보수 원칙:**
- 볼린저 밴드 계산 불가(데이터 부족)이면 0점
- MA5 계산 불가이면 0점

### 3.4 조건 3: 거래량 확인 (30점)

**충족 조건 (시장별 차이):**

#### US / KR 공통: 거래량 서프라이즈
```
Volume_curr > Volume_MA20 × 1.5
```

- 평균 대비 50% 이상 거래량 증가 = 시장 참여자의 관심 유입
- 반등에 "돈이 따라오고 있다"는 확인

#### KR 추가 조건 (거래량 OR 수급):
```
(Volume_curr > Volume_MA20 × 1.5)
OR
(foreign_net_buy_value > 0 AND institution_net_buy_value > 0)
   → 외인+기관 동시 순매수 (당일)
```

- KR 시장은 수급 데이터가 있으므로, 외인+기관 동시 매수도 같은 가중치 부여

**보수 원칙:**
- 거래량 데이터 20일 미만이면 0점
- 거래량이 0이면 0점 (장 외 데이터 오류 방지)

## 4. 시장 환경 필터 (Tier 2 Kill-Switch)

Tier 1과 동일한 Kill-Switch를 공유하되, **추가 완화 필터**를 적용합니다:

### 4.1 공유 Kill-Switch (Tier 1과 동일)

| 조건 | 동작 |
|------|------|
| VIX > 30 | Tier 2 전면 차단 |
| US10Y change_pct > 3% | Tier 2 전면 차단 |
| KRWUSD > BB upper | Tier 2 전면 차단 |

### 4.2 Tier 2 추가 필터

| 조건 | 동작 | 이유 |
|------|------|------|
| VIX > 25 | 점수에 ×0.7 패널티 | 불안정 시장에서 단기 반등 신뢰도 낮음 |
| 당일 시장 지수 -3% 이상 급락 | Tier 2 당일 차단 | 공황 중 개별 반등은 데드캣 바운스 가능성 |

**"VIX 25~30 경고 구간"에서의 차이:**
- Tier 1: 경고 문구만 추가, 점수 유지 (장기 진입이라 VIX 25~30이 오히려 기회)
- Tier 2: 점수 × 0.7 패널티 (단기 반등의 신뢰도가 VIX 높을 때 떨어지므로)

## 5. 출구 전략 (Exit Rules)

Tier 2의 핵심은 **명확한 출구**입니다. 진입보다 출구가 더 중요.

### 5.1 목표가 (Take Profit)

```
목표가 = min(MA20, 진입가 × 1.10)
```

- MA20까지 반등하거나, +10% 중 먼저 도달하는 쪽
- 욕심부리지 않음 — "평균으로의 회귀"만 취함

### 5.2 손절가 (Stop Loss)

```
손절가 = 진입가 - ATR(14) × 1.5
```

- Tier 1(×2.0)보다 타이트 (단기 트레이드이므로)
- ATR 없으면 진입가 × 0.95 (5% 고정 손절)

### 5.3 타임컷 (Time Stop)

```
진입일로부터 15거래일 내 목표가/손절가 미도달 시 → 전량 청산
```

- 15일 = 약 3주 (단기 스윙의 최대 보유 기간)
- "움직이지 않는 종목"에 묶이는 것을 방지

### 5.4 트레일링 스탑 (선택적)

```
진입 후 최고가 대비 -ATR(14) × 1.0 이탈 시 → 전량 청산
```

- +5% 이상 수익 발생 후, 수익을 지키기 위한 후행 손절
- 구현 복잡도가 있으므로 V1에서는 생략 가능 (V2에서 추가)

## 6. 중복 방지 로직

### 6.1 Tier 1 발동 종목 제외

```python
if ticker in tier1_alerted_last_60_days:
    skip  # Tier 1이 이미 커버 중
```

### 6.2 동일 종목 재알림 쿨다운

```
동일 종목 Tier 2 알림 후 10거래일간 재알림 차단
```

- "한 번 잡았으면 결과를 보고 다시 판단"
- 연속 알림으로 인한 피로도 방지

### 6.3 일일 알림 상한

```
하루 최대 3종목까지만 Tier 2 알림 발송
```

- 우선순위: 점수 높은 순
- 너무 많은 알림 = 노이즈 = 시스템 가치 하락

## 8. 백테스트 검증 기준

Tier 2를 배포하기 전에 반드시 백테스트로 검증해야 할 지표:

| 지표 | 목표 | 불합격 기준 |
|------|------|-------------|
| 승률 (Win Rate) | ≥ 55% | < 50% |
| 평균 수익 / 평균 손실 (Reward/Risk) | ≥ 1.2 | < 1.0 |
| 최대 연속 손실 (Max Consecutive Loss) | ≤ 5회 | > 7회 |
| 월 평균 알림 수 | 3~10회 | > 15회 (노이즈) 또는 < 1회 (무용) |
| 타임컷 비율 | ≤ 30% | > 50% (방향성 없는 구간 진입 많음) |
| 평균 보유 기간 | 5~10일 | > 15일 (스윙이 아닌 포지션 트레이딩) |

### 8.1 백테스트 기간

- **In-sample**: 2020-01 ~ 2024-12 (코로나 폭락, 금리 인상기, 반등기 모두 포함)
- **Out-of-sample**: 2025-01 ~ 2026-03

### 8.2 백테스트 필수 시나리오

| 시나리오 | 기대 동작 |
|----------|-----------|
| 2020-03 코로나 폭락 | VIX > 30 Kill-Switch → 알림 차단 |
| 2020-04~05 반등기 | VIX 하락 후 대량 알림 발생 |
| 2022-01~10 금리 인상 하락기 | 과매도 진입 빈번 → 손절 비율 확인 |
| 2023 횡보장 | 알림 빈도 적절한지 확인 |
| 2024 강세장 | 알림 거의 없음 (과매도 구간 희소) |

## 9. 추가 데이터 수집 필요 사항

기존 DataCollector에서 **추가 수집 불필요**. 모든 지표는 기존 데이터에서 파생 계산 가능:

| 지표 | 데이터 소스 | 계산 위치 |
|------|-------------|-----------|
| MA5 | close_value × 5일 | SwingAnalyzer Lambda |
| 볼린저 밴드 | close_value × 20일 | SwingAnalyzer Lambda |
| Volume MA20 | volume_value × 20일 | SwingAnalyzer Lambda |
| RSI prev_prev | DynamoDB 3일 전 조회 | SwingAnalyzer Lambda |
| ATR(14) | OHLC × 15일 | 기존 `_calc_atr()` 재사용 |

**→ DataCollector 수정 없음. 인프라 추가 비용 = Lambda 실행 비용만.**

## 10. 아키텍처 (배포 구조)

```
EventBridge (장 중, Tier 1과 동일 스케줄 또는 별도)
      │
      ▼
SwingAnalyzer (새 Lambda)
  ├─ DynamoDB 조회 (기존 테이블 Read-Only)
  ├─ 시장 환경 필터 (Kill-Switch 공유)
  ├─ 파생 지표 계산 (MA5, BB, Vol MA20)
  ├─ Tier 2 스코어링
  ├─ 중복 방지 (DynamoDB 쿨다운 테이블)
  └─ Score >= 70 → AlertingEngine 호출 (기존 Lambda 재사용 또는 별도)
```

### 10.1 새로 필요한 리소스

| 리소스 | 용도 |
|--------|------|
| Lambda `hcses-swing-analyzer-kr` | KR 스윙 분석 |
| Lambda `hcses-swing-analyzer-us` | US 스윙 분석 |
| DynamoDB Table (또는 기존 테이블에 GSI) | 쿨다운 기록 (`ticker + last_alert_date`) |
| Secrets Manager | Discord 웹훅 (별도 채널이면 별도 시크릿) |
| EventBridge Rule | 스케줄 트리거 |

### 10.2 추가 월 비용 예상

| 항목 | 비용 |
|------|------|
| Lambda (SwingAnalyzer KR/US) | ~$0.40 |
| DynamoDB (쿨다운 테이블) | ~$0.02 |
| **합계** | **~$0.42/월 추가** |

## 11. 티커 선정 전략

### 11.1 Tier 2에 적합한 종목 특성

Tier 1과 동일한 종목 리스트를 사용하되, 아래 특성을 가진 종목이 Tier 2에서 더 자주 트리거됩니다:

| 특성 | 이유 |
|------|------|
| 베타 > 1.2 | 시장 대비 변동성 큼 → 과매도 구간 진입 빈번 |
| ATR / Close > 3% | 일일 변동폭 큼 → 단기 반등 폭 큼 |
| 유동성 풍부 | 거래량 서프라이즈 시그널이 유의미 |

### 11.2 Tier 2에서 주의할 종목

| 종목 | 주의점 |
|------|--------|
| UNH, BMY (방어주) | RSI 35 이하 진입 자체가 드물 → 트리거 희소 |
| META (빅테크) | 모멘텀 강해서 과매도 시 빠르게 회복 → 진입 타이밍 까다로움 |
| T (통신) | 변동성 작음 → +10% 목표가 도달 어려움 → 타임컷 빈발 가능 |

### 11.3 종목별 파라미터 튜닝 (V2)

V1에서는 전 종목 동일 파라미터. 백테스트 결과에 따라 V2에서:
- 고변동 종목 (MU, AMD, FCX): 목표가 +12%, 손절 ATR×1.5
- 저변동 종목 (T, UNH): 목표가 +7%, 손절 ATR×1.2
- 또는 해당 종목을 Tier 2 대상에서 제외

## 12. 리스크 관리 원칙

### 12.1 단일 종목 한도

```
종목당 투자금 = 총 예산의 5% (Tier 1의 10%보다 작음)
```

- 단기 트레이드는 확신이 낮으므로 사이징 축소

### 12.2 동시 포지션 제한

```
최대 동시 Tier 2 포지션 = 3개
```

- 4번째 알림이 오면 무시 (또는 기존 포지션 정리 후 진입)
- 집중도 유지, 관리 가능한 범위

### 12.3 주간 손실 한도

```
주간 Tier 2 누적 손실 > 총 예산의 5% → 해당 주 Tier 2 중단
```

- 연속 손절 발생 시 시장 환경이 Tier 2에 불리한 상태
- "쉬는 것도 트레이딩"
